# Coding Rules & Guidelines (Antigravity & Claude Code)

このプロジェクト `sleep-tracker` では、以下のコーディング規則および開発ガイドラインを厳格に適用します。
開発を行うすべてのAIアシスタント（Antigravity、Claude Code、およびその他）は、これらのルールを遵守してください。

---

## 共通ルール

### 1. AIアシスタント間のルール同期 (Antigravity & Claude Code)
- 本ファイル (`.agents/AGENTS.md`) は、このプロジェクトにおけるエージェント共通のコーディング規則です。
- **片方のAIアシスタントがこのルールを変更・修正した場合、必ずもう片方のAIアシスタントもその変更を認識し、このルールファイルに同期して記述を更新しなければなりません。**
- 開発作業を行う際は、常にこのファイルの最新版を確認し、それに従ってください。

### 2. ファイルサイズ制限
- **すべてのソースコードファイルは500行以下** でなければなりません。
- 1つのファイルの記述量が500行を超えそうな場合は、クラスや関数を適切にモジュール（ヘッダーファイルや別モジュール）に分割してください。

### 3. ファイルヘッダーコメント
- すべてのソースコードファイル（C++, Python等）の**先頭10行以内**に、そのファイルの情報をまとめたコメントを記述してください。
- 記述項目:
  1. ファイル名
  2. ファイルの概要（このファイルが何をするものか）
  3. 作成日
  4. 主要なクラス・関数
  5. 依存関係や特記事項

### 4. フォルダ毎のドキュメント設置（階層的ナビゲーション）
- プロジェクト内の各フォルダ（ディレクトリ）に、統一した名称のマークダウンファイル `README.md` を設置します。
- この `README.md` には以下を記述してください：
  - フォルダの役割・概要
  - フォルダ内のファイル・サブフォルダの一覧と、それぞれで何が行われているかの説明
  - LLM（AIエージェント）がこのディレクトリ構造を階層的に辿るためのナビゲーション情報

## 技術別ルール

### Python (uv)
- パッケージ管理には `uv` を使用します。
- 依存関係の追加は `uv add <package>` を用いて `pyproject.toml` に追記してください。
- コードは PEP 8 に準拠し、適切な型ヒントを記述してください。

### C++
- Windows API を用いるため、Windows環境でビルド可能なように構成します。
- メモリリークのないよう、スマートポインタや RAII を活用してください。
- バックグラウンドで極めて軽量に動作する（CPU・メモリリソースを極力消費しない）ように設計してください。
